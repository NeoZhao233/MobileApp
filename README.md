# Obsidian Lite for HarmonyOS

一个面向 HarmonyOS 的轻量笔记应用原型，灵感来自 Obsidian。目标是用更简单的交互复刻核心体验：笔记库、Markdown 阅览与编辑、标签、搜索、双向链接、反链与轻量图谱。

## 项目定位

- 平台：HarmonyOS（ArkUI + Stage 模型）
- 存储策略：本地优先（当前为原型阶段）
- 产品目标：先做可演示原型，再迭代到 MVP 可用版本

## 当前已实现功能

- 笔记库列表
  - 卡片式展示笔记
  - 支持按关键词搜索（标题、正文、标签）
  - 支持标签筛选
- 笔记阅览与编辑
  - 点击笔记进入该笔记详情
  - 默认先阅览 Markdown
  - 用户可主动切换到编辑模式
- Markdown 轻量渲染
  - 标题（一级到三级）
  - 列表
  - 引用块
  - 行内加粗、行内代码、双链文本
- 双向链接与反链
  - 自动提取正文中的双链
  - 显示当前笔记的反向链接来源
- 图谱 Lite
  - 节点与连线关系展示（轻量版）
- 创建笔记流程
  - 顶部按钮进入创建页
  - 填写标题、正文、标签后创建
- 响应式布局
  - 支持紧凑模式与三栏模式切换
- 中文界面与视觉主题
  - 暖纸张背景
  - 轻阴影卡片
  - 橙色强调色

## 交互说明

- 在笔记库中点击任意笔记，会显示该笔记详细内容
- 详情默认是阅览态，不会直接进入编辑
- 点击“编辑笔记”后进入编辑态，完成编辑后返回阅览态
- 新建笔记需点击按钮进入独立创建页

## 技术栈

- ArkTS
- ArkUI（声明式 UI）
- HarmonyOS Stage 应用模型

## 目录结构概览

- entry：主应用模块与页面代码
- AppScope：应用级配置与资源
- hvigor：构建配置
- oh-package.json5：依赖与工程配置

## 后续计划（MVP 路线）

- 本地持久化存储（重启不丢数据）
- 链接点击跳转（含不存在笔记处理）
- 图谱交互增强（缩放、聚焦、邻居视图）
- 分类体系（独立于标签）
- 搜索增强（高亮、相关性排序）
- Markdown 能力扩展（代码块、任务列表、表格等）
