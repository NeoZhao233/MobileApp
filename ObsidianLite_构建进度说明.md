# Obsidian Lite 构建进度说明

## 当前阶段

- 当前状态：原型可演示（Prototype）
- 目标状态：MVP 可持续使用（本地数据不丢失、关键交互闭环）

## 已实现的核心功能

### 1. 笔记库（Note Library）
- 笔记卡片列表展示
- 选中笔记后进入该笔记内容阅览
- 新建笔记入口（通过按钮进入独立创建页）

### 2. 笔记阅览与编辑流程
- 默认先阅览 Markdown 内容
- 用户主动点击“编辑笔记”后进入编辑态
- 可在编辑态完成标题、正文、标签修改
- 编辑完成后可返回阅览态

### 3. Markdown 轻量渲染（Preview Lite）
- 已支持：
  - 标题（#、##、###）
  - 列表（-）
  - 引用（>）
  - 行内加粗（**text**）
  - 行内代码（`code`）
  - 双链文本（[[link]])

### 4. 标签与筛选
- 标签展示
- 标签筛选
- 对当前笔记添加新标签

### 5. 全文搜索（基础）
- 支持按关键词检索：
  - 标题
  - 正文
  - 标签

### 6. 双向链接相关能力（Lite）
- 正文中自动提取 [[...]] 链接
- 生成轻量图谱边关系
- 反向链接（Backlinks）面板展示

### 7. 图谱交互视图（Graph Interactive）
- 节点展示（笔记标题）
- 连线关系文本展示
- 双向关系标注
- 图谱画布支持拖拽浏览（横向/纵向）
- 支持图谱缩放（Zoom In/Out）
- 支持节点点击聚焦（邻居视图）与一键取消聚焦

### 8. UI 与布局
- 中文界面
- 暖纸张 + 卡片视觉风格
- 沉浸式阅读与编辑状态无缝切换
- 创建页覆盖层流程（点击后出现）

### 9. 本地持久化存储
- 基于 AppStorage 与 PersistentStorage 的本地核心数据持久化机制
- 实现了应用加载时读取以及修改后的实时自动保存

### 10. 双链无缝跳转与游历
- 点击 [[...]] 链接直接切换为对应目标笔记的内容闭环
- 支持识别不存在链接，并在点击时自动无缝创建关联空笔记
- 解决了 ArkUI 状态刷新、组件复用与列表渲染的多个底层兼容性与排障体验问题

## 未实现或仅部分实现的核心功能

### P1（MVP 强建议）

#### 4. 分类体系（Category）
- 当前以标签为主，尚无独立分类/笔记本模型

#### 5. 笔记管理闭环
- 缺少删除、排序、置顶、最近访问等能力

#### 6. 搜索增强
- 缺少命中高亮、相关性排序、模糊匹配

### P2（体验增强）

#### 7. Markdown 扩展
- 缺少代码块、任务列表、表格、图片等更完整语法支持

#### 8. 稳定性与可观测性
- 缺少存储异常处理、损坏恢复与日志化机制

## 结论

- 当前阶段 **P0 目标已全部完成**，Obsidian Lite 已成功跨过原型的基础演示，建立起了真正的本地数据与“无缝创建与双链跳转”的操作闭环。
- 距离 MVP 的关键差距如今在 P1 强推荐特性中：
  - 分类（Category/Notebook）机制与卡片管理操作（删除/置顶/排序）
  - 搜索高亮与模糊匹配的完善
